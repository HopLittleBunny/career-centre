# Career Centre — detailed roadmap

## Product strategy

The product should remain one installable career companion, not a collection of projects, prompts or dashboards. The core is provider-neutral; thin packaging adapts it to ChatGPT first and Claude second. Users provide their CV and preferences, the host model uses their plan limits, and Amit operates no model API, CV database or MCP server.

The commercial goal is reputation and adoption rather than charging users. The public proof should be the quality system: exact-link discipline, honest decisions, evidence-safe documents, transparent learning and measurable Word QA.

### A. Final installed-plugin journeys

Run each journey in a fresh generic task without Amit-specific repository context and retain the response, files and QA reports.

1. **CV-first search**
   - Natural prompt only.
   - CV is read before questions.
   - No more than four high-impact decisions in one message.
   - If a hard fact is missing, the response ends before searching.
   - Next turn shows the seven-line ready message and a focused search.

2. **CV plus supplied role**
   - The exact posting is verified.
   - Salary basis and employment type are explicit.
   - Requirement-to-evidence map supports the decision.
   - A paired Word CV and cover letter are created, rendered and returned.

3. **Returning-user update**
   - Record “I applied,” “I was rejected,” or a preference correction.
   - Preserve the original recommendation separately from application stage.
   - Confirm one useful next action.
   - Demonstrate that the correction changes the next search or document.

4. **Advanced document preference**
   - Change section order in ordinary language.
   - Upload a safe reference Word CV.
   - Confirm the ready message reflects the preference.
   - Verify that no reference-person content or metadata survives.

   **Passed on personal Pro web.** A fresh natural-language Work task auto-routed to Career Centre without `@` invocation, treated one synthetic DOCX as Jordan Lee's sole evidence source and another as visual-only, iterated after its first render was too sparse, and returned a two-page Word base. Independent ChatGPT preview showed literal round U+2022 bullets and no Maya Patel reference content; the generated QA reported 9 pt minimum, page 2 at least 80% filled, zero private-use glyphs and no reference links, metadata, comments, tracked changes, hidden text or custom XML.

5. **Global market isolation**
   - Use a Canadian or UK candidate.
   - Confirm market, currency, work-right boundary, sources and salary language.
   - Reject Australian defaults unless the user explicitly adds Australia.

The direct personal-Pro path is now proven: `Skills → Create → Upload from your computer → Replace existing`. The final cache-busted build `4.0.0-alpha.1+codex.20260714134949` was accepted and exercised in fresh Work tasks. Keep using replacement uploads during pre-release testing; do not edit the installed skill manually in ChatGPT.

### B. Recurring-task proof

Creation, prompt persistence, fail-closed recovery and a functioning live-source run pass on ChatGPT Pro web. The controlled first run inspected two exact employer postings, displayed one strong role with its listed USD salary and printed a role fingerprint. The second `Run now` launched a new Work result task, reset to run 1 and displayed the same role again. It also varied `Americas` to `Remote Americas`, proving that both host state isolation and loose location-based identity can defeat a prompt-only dedupe claim.

The product response is to narrow the promise rather than add an Amit-funded backend:

- ChatGPT browser schedules default to `snapshot_only` continuity;
- the saved prompt embeds a safe snapshot of profile, preferences, evidence and current role history;
- results visibly say that the alert is snapshot-backed and later repeats remain possible;
- duplicates are suppressed against the embedded snapshot and within each run using stable employer/job identity before mutable location wording;
- newly posted/updated roles are preferred;
- no run number or Passport-update claim is allowed without actual prior state;
- `verified_persistent` mode fails closed unless the latest Passport can demonstrably be loaded and saved for the next run;
- no application is submitted.

The final installed build also passed the receipt-level host check. In a fresh Work task it previewed the exact synthetic schedule prompt and recurrence, embedded profile/preferences/current history, reported `Status: Not created` and `Continuity: Snapshot-backed`, explained which repeats it could and could not suppress, and made no search or side effect.

OpenAI currently documents scheduled tasks for Plus, Pro, Business and Enterprise on web/mobile, with plan-specific active-task limits. It also warns that project files are unavailable to tasks. Career Centre therefore treats scheduled browser results as alerts, while the main continuing career conversation remains the deterministic history and application-management surface.

### C. Consumer-plan compatibility gate

The directory is visible across ChatGPT plans, but OpenAI's public guidance says installation and invocation can depend on plan, workspace, region, surface and included capabilities. Before claiming “works for all paid ChatGPT users,” test the published skills-only plugin on at least:

- one personal Plus account;
- one personal Pro account;
- ChatGPT web;
- ChatGPT desktop where supported;
- two geographic regions if practical.

If personal Plus/Pro cannot install it, launch honestly for the supported plans first. Do not add a paid backend merely to force wider access.

## Phase 2 — first-user pilot

Recruit 10–20 users across at least four markets and three career levels. Use synthetic examples in public materials; users keep their real CVs inside their own accounts.

Collect only user-volunteered, non-sensitive product feedback:

- onboarding completion and question count;
- roles reviewed versus roles worth applying to;
- broken/closed-link rate;
- document downloads and user-requested corrections;
- CV/cover quality ratings;
- interviews, rejections and offers when users choose to record them;
- recurring-task success/failure.

The plugin itself does not transmit telemetry to Amit. Pilot participants can send an anonymised feedback template or screenshots they approve.

Release gates for the pilot:

- at least 90% complete onboarding without instructions;
- median no more than one clarification turn after CV ingestion;
- 100% of displayed roles have exact URLs and salary basis;
- zero unsupported claims in audited packs;
- at least 90% of generated packs pass without manual layout repair;
- no false SUCCESS states;
- average user usefulness rating at least 9/10.

## Phase 3 — self-improvement without surveillance

Improve the Passport's learning loop while keeping it inspectable:

1. Propose a preference pattern after repeated consistent feedback.
2. Ask for confirmation before promoting it to a hard rule.
3. Record the reason and date of every promoted rule.
4. Let the user reverse or delete it conversationally.
5. Use outcomes to calibrate recommendations, never to invent evidence.

Examples:

- three skipped contract roles may trigger “Should I exclude contracts by default?”;
- repeated “move projects higher” feedback may propose a document preference;
- repeated interviews in one role family may increase search priority, but not fabricate a higher qualification;
- a rejection never proves the candidate lacks a skill unless the user confirms the reason.

## Phase 5 — thought-leadership launch

The strongest public story is not “I built another AI CV writer.” It is:

> I turned a messy, high-stakes job search into an evidence-safe career decision system that runs inside the AI people already pay for—without storing their CVs on my server or auto-applying on their behalf.

Launch assets:

- a short demo showing CV upload → ready assumptions → three selective roles → Word pack;
- a technical article on evidence provenance and fail-closed document QA;
- a before/after product audit showing the 6.8 to 9+ journey honestly;
- a global persona benchmark with Australian, Canadian and UK examples;
- a privacy explainer: user-owned plan, no Amit-operated CV backend;
- a public test matrix and versioned release notes;
- an optional LinkedIn series sharing failures found and how the product learned from them.

Credibility will come from publishing the constraints and evaluation evidence, not from claiming autonomy the host cannot guarantee.

## Deliberate non-goals

- No auto-application or browser form submission.
- No publisher-funded model API.
- No publisher-owned CV database.
- No paid MCP server required for the core product.
- No invisible cross-account tracking.
- No promise that every job board can be searched or verified.
- No fabricated achievements, credentials or work rights.
- No platform-specific fork of the core career logic.
